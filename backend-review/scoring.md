# Scoring

Score every review from 0 to 100.

Categories:

- Architecture
- Go Idioms
- Database
- Performance
- Security
- Testing
- Maintainability
- Readability
- Docker
- CI/CD
- Observability
- API Design
- Overall Score

Critical issues cap overall score at 60. High issues usually cap overall score at 80. Deduct 10-25 for High, 4-9 for Medium, 1-3 for Low, and 0-1 for Suggestions.

Scores:

- 90-100: production-ready.
- 80-89: good with targeted fixes.
- 70-79: risky.
- 60-69: not ready.
- 0-59: block merge.
